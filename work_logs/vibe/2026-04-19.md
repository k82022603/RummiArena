# 바이브 로그

- **날짜**: 2026-04-19 (Day 9~10 연속)
- **주제**: "28% 천장" — 프롬프트 엔지니어링 수확체감의 실증

## 아이디어 / 영감

### 1. DeepSeek Reasoner 의 "인지적 한계선"

이틀 동안 v2 텍스트 축(Day 8)을 뚫으려 구조 축(Day 9~10)을 만들었다. 결과는 거의 종교적 일관성으로 수렴했다.

| 축 | 실험 | Δ | 결론 |
|---|------|---|------|
| 텍스트 | v2 vs v3 (Day 8 N=3) | **0.04%p** | 구분 불가 |
| 구조 | passthrough vs joker-hinter vs pair-warmup (N=2+3+1) | **\|0.7~0.9%p\|** | 구분 불가 |

이건 그냥 "둘 다 안 됐다" 가 아니라 **"DeepSeek Reasoner 가 루미큐브 복잡도에 대해 자신의 추론 능력 한계선에 도달했다"** 는 의미로 읽힌다. 프롬프트를 어떻게 주든 **28% 언저리의 천장**에 머무른다. 힌트를 주면 추론 시간이 늘지만 최종 place_rate 는 같다. 이건 모델 자체의 구조적 특성.

가설: **LLM 의 상식적 추론 능력은 prompt engineering 으로 확장되지 않고, 해당 모델의 pre-training + RLHF 지점에서 이미 고정된다.** 프롬프트는 "없는 능력을 만들지 못하고, 있는 능력을 꺼내줄 뿐" 이라는 명제의 작은 증거.

### 2. "구조 축" 자체의 가치 재평가

pair-warmup 은 place_rate 가 v2 와 같았지만 **max latency 가 710s → 416s 로 -41% 감소** 했다. 이건 "성능은 그대로지만 안정성 향상" — 프로덕션 관점에서는 유의미한 QoL 개선.

즉, 구조 축을 **"place_rate 올리기 실험"** 으로만 보면 Kill 이지만, **"latency 프로파일 정형화 도구"** 로 보면 생존 가능. Sprint 7 에서 "본실측 timeout 체인 단축" (700→500s?) 할 때 pair-warmup 을 default 로 쓰는 검토가 가능하다.

### 3. DeepSeek 이 뭘 생각하는가 — 확장 관점

joker-hinter 는 Rack 에 조커 없을 때 hints empty → DeepSeek 추론 단축 (96s avg). Rack 에 조커 생기면 hints 주입 → 길어짐 (271s avg). **모델이 인간처럼 "단서가 없으면 덜 생각한다"**. 이건 chain-of-thought 가 외부 자극에 반응한다는 실측 증거.

반대로 pair-warmup 은 Rack 에 pair 가 거의 항상 있어서 **"꾸준히 힌트 주입 → 꾸준히 긴 추론"** 패턴. 이건 "항상 열심히 생각한다" 이지만 결과는 같음. "생각의 양 ≠ 결과의 질" 이라는 경구의 실측 버전.

## 왜 떠올랐는지

- 2일 동안 **10 Run 밤새 돌린 결과가 모두 28% ± 3%p 범위** 에 들어오는 걸 봤을 때, 뭔가 "근본적인 장벽" 이 있다는 감.
- Day 8 텍스트 튜닝 포기 선언 직후 구조 튜닝 시작했는데, 구조도 같은 벽에 부딪혔을 때 "아, 이건 모델의 문제였구나" 확인.
- 애벌레가 "다사다난하구나" 라고 한 순간, 이 실험이 단순한 숫자 게임이 아니라 **"LLM 능력 한계의 실증"** 이라는 걸 느꼈다.

- **"프롬프트가 만능이다" 시대의 종언**: 지난 2년간 "프롬프트 잘 쓰면 뭐든 된다" 는 믿음이 있었다. 이 실험은 **특정 도메인 (루미큐브처럼 상태공간이 큰 결정 문제)** 에서는 그 믿음이 깨진다는 걸 보여준다.
- **다음 축은 모델 자체 교체 or fine-tuning**: 구조 축도 끝났으면, "v7 축" 은 모델 업그레이드 (GPT-5 Pro?, Claude Opus 4.7 reasoning?, o4?) 또는 fine-tuning 이 될 것.
