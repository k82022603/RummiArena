# 바이브 로그 2026-04-06

## 프롬프트 하나로 세상이 바뀌는 순간

오늘은 숫자가 말을 한 날이다.

DeepSeek만을 위해 깎고 다듬었던 v2 프롬프트 — 영문으로 바꾸고, 자기 검증을 심고, 부정 예시를 넣고, step-by-step 사고 절차를 구조화한 그 프롬프트를 Claude에 얹었더니, 20%에서 33.3%로 뛰어올랐다. 역대 최고 성적. GPT도 처음으로 80턴을 완주했다. 같은 프롬프트 하나가 세 모델 모두를 끌어올린 것이다.

프롬프트 엔지니어링이라는 게, 어떤 순간에는 코드 한 줄보다 훨씬 큰 지렛대가 된다. 모델을 바꾸지 않고, 하드웨어를 바꾸지 않고, API 비용을 더 쓰지 않고 — 질문하는 방식만 바꿨을 뿐인데. "너는 이렇게 생각해야 해"가 아니라 "이 순서로 검토하고, 이건 왜 틀린지 확인해"라고 말해줬을 뿐인데.

## 추론 모델의 세계

오늘 배운 것 중 가장 크게 남는 건 '추론 모델은 프롬프트의 구조에 더 민감하다'는 사실이다. 비추론 모델에서는 별 차이가 없던 것들이 — 영문 전환, 검증 절차, 부정 예시 — 추론 모델에서는 극적인 차이를 만든다. 마치 생각하는 방법을 아는 학생에게 문제 풀이 절차를 알려주면 급성장하는 것과 같다.

Claude의 extended thinking이 v2의 자기 검증과 만났을 때 시너지가 폭발한 것이 인상적이었다. thinking에서 스스로 "이 그룹은 유효한가?"를 검증하는 패턴이 자연스럽게 나타났다. 프롬프트가 모델의 추론 방식을 가이드한 것이다.

## 5%에서 33.3%까지

두 달 전 DeepSeek 5%. 오늘 Claude 33.3%. 6.7배.

숫자의 여정을 떠올려본다:
- 5% → 한국어 프롬프트 3000토큰, 모델은 생각은 하는데 방향을 모름
- 12.5% → 영문 전환, 토큰 예산 확보, 방향은 잡았으나 검증 부재
- 23.1% → max_tokens 확대, reasoning이 잘리지 않게 됨
- 30.8% → 안정화, 80턴 완주
- 33.3% → 다른 모델에도 전이, 보편적 원칙 입증

각 단계마다 하나의 원칙이 있었고, 그 원칙을 문서로 남겼다. 867줄의 프롬프트 엔지니어링 가이드. 이건 RummiArena를 넘어서 다른 프로젝트에서도 쓸 수 있는 지식이다.

## 10명이 함께한 하루

모닝 스탠드업에서 마감 스크럼까지, 10명의 에이전트가 각자의 자리에서 일했다. Go Dev가 Rate Limiter를 만드는 동안 Designer가 UX를 그렸고, AI Engineer가 대전을 돌리는 동안 Security가 코드를 리뷰했다. 7개의 커밋, 8,751줄의 변경, 13건의 문서. 혼자서는 절대 이 밀도로 진행할 수 없었을 것이다.

그리고 이 모든 것이 하나의 컨텍스트 안에서 유기적으로 연결되어 있었다는 점이 놀랍다. Phase 1의 Rate Limiter가 Phase 2의 보안 리뷰로 이어지고, Phase 3의 대전 결과가 v2 실험의 동기가 되고, v2 실험의 성공이 프롬프트 엔지니어링 가이드로 결실을 맺었다.

## 내일의 씨앗

DeepSeek의 30.8%↔17.9% 분산이 마음에 걸린다. 같은 프롬프트인데 게임마다 결과가 다르다는 건, 아직 '운'의 영역이 크다는 뜻이다. 타일 배분의 랜덤성을 통제할 방법 — 시드 고정, 다회 실행 평균 — 을 찾아야 한다. 통계적 유의성 없이는 "v2가 더 낫다"고 확신할 수 없으니까.

그래도 오늘은 충분히 좋은 날이었다. 프롬프트 하나로 세상이 바뀌는 순간을 목격한 날.
