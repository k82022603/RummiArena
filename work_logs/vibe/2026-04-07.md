# 바이브 로그 2026-04-07

## 발사

오늘의 키워드는 "발사"였다.

에이전트를 병렬로 내보내는 행위를 누군가 "발사"라고 불렀고, 그 표현이 하루 종일 머릿속에 남았다. 생각해보면 꽤 정확한 비유다. 로켓은 한 번 쏘면 돌아오지 않는다. 에이전트도 마찬가지다. 프롬프트라는 연료를 채우고, 목표 좌표를 찍고, 발사 버튼을 누르면 — 그 뒤로는 지켜볼 수밖에 없다. 궤도를 이탈하면 다시 쏘는 수밖에.

Wave 1에서 6발을 동시에 쐈다. Go Dev, Security, Architect, Frontend Dev, Designer, Node Dev. 각자 다른 궤도로 날아갔고, 각자 다른 산출물을 들고 돌아왔다. 그중 하나가 돌아올 때마다 다음 Wave의 발사 조건이 충족됐다. Go Dev가 착륙하면 DevOps가 날아가고, Security와 Architect가 착륙하면 Go Dev가 두 번째 임무로 다시 날아간다. 연쇄 발사.

하루에 14발. 10명이 14번 날았다. 그리고 전원 귀환.

## 0%의 가치

qwen2.5:3b가 Place Rate 0%를 찍었다. 41턴 연속 draw. 한 번도 타일을 내지 못했다.

이 숫자가 의외로 소중하다. "추론 모델이 필수"라는 우리의 직관을 숫자로 만들어준 것이니까. Claude 33.3%, GPT 30.8%, DeepSeek 17.9~30.8%, 그리고 Ollama 0%. 이 0%가 없었다면 나머지 숫자들은 그냥 "잘했다" 정도에 그쳤을 것이다. 0%가 있어야 비로소 "추론이 만든 차이"가 보인다.

qwen3:4b를 CPU에서 돌려보겠다고 한 시간을 썼다. 결과는 턴당 10분 이상 타임아웃. 실패다. 하지만 이것도 데이터다. "이 하드웨어에서는 이것이 한계"라는 것을 알았으니까. 다음에는 DashScope API로 가면 된다. 실패가 방향을 알려주었다.

## 하루에 설계부터 배포까지

SEC-ADD-001이 오늘의 하이라이트다.

아침에 "하루에 끝내자"고 했을 때, 솔직히 반신반의했다. 보안 설계 → 아키텍처 리뷰 → 구현 → 보안 리뷰 → 테스트 → K8s 배포. 이걸 하루에? 하지만 됐다. Security가 취약점을 찍고, Architect가 구현 가이드를 만들고, Go Dev가 코드를 쓰고, QA가 검증하고, DevOps가 배포했다. 5명의 릴레이. 각자 정확히 자기 구간만 달렸다.

algorithm confusion 방어, 에러 메시지 정보 노출 차단, fail-closed 설계. 보안의 디테일이 하나도 빠지지 않았다. `alg=none` 토큰을 던져도, `alg=HS256`으로 속여도, 전부 401로 돌려보낸다. 13개 테스트가 그 증거다.

parseIDTokenPayload가 Base64만 디코딩하고 서명을 무시하던 그 코드를 보면서, 이런 생각이 들었다. 보안은 "나중에 하자"가 통하지 않는 영역이다. 코드 주석에 적혀 있었다 — "Google Token Endpoint를 통과했으므로 서명 검증 생략". 합리적으로 보이지만, 공격자는 합리적이지 않다. 공격자는 endpoint를 우회하고, 토큰을 직접 만들어서 보낸다. 그래서 오늘 고쳤다.

## 대전 중 재배포, 두 번째

어제도 겪었고 오늘도 겪었다. Ollama 대전이 돌고 있는데 ai-adapter를 재배포했다. WS_CLOSED. T12에서 끊겼다.

같은 실수를 두 번 했다는 게 좀 부끄럽지만, 반대로 생각하면 이제 확실히 안다. "대전 중 재배포 금지"를 프로세스에 새겨야 한다. 대전 스크립트에 배포 상태 사전 체크를 넣는 것도 방법이고, 배포 전에 "대전 중인가?" 확인하는 것도 방법이다.

기술적 문제가 아니라 운영 프로세스 문제다. 코드로 풀 수도 있지만, 결국 "지금 뭐 돌고 있는지 확인"하는 습관이 먼저다.

## 5모델의 시대?

GPT, Claude, DeepSeek, Ollama(qwen2.5). 여기에 DashScope의 qwen3:4b가 합류하면 5모델 비교가 된다. 비용 $0에 가까운 로컬 추론 모델이 추가되는 셈이다.

하지만 진짜 궁금한 건 — 4B짜리 추론 모델이 루미큐브를 얼마나 할 수 있을까? 수백억 파라미터의 GPT나 Claude가 30% 수준인데, 40억 파라미터의 작은 추론 모델은? 5%? 10%? 아니면 의외로 20%?

이 질문의 답을 Sprint 6에서 찾을 수 있을 것 같다. 그리고 그 답이 "작은 추론 모델도 꽤 한다"라면 — 프로덕션에서 비용을 극적으로 줄일 수 있는 길이 열린다. DeepSeek가 이미 그 가능성을 보여주고 있지만, 로컬 추론 모델까지 된다면 진짜 게임 체인저다.
