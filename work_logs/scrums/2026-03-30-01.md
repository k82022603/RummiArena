## 논의 사항

### 1. Git 히스토리 시크릿 정리 (bfg) 타이밍
- **DevOps**: bfg + force push는 모든 로컬 clone을 리셋해야 함. 오늘 중 실행 제안.
- **PM**: 다른 작업 브랜치 없으므로 즉시 가능. 정리 후 전원 `git fetch --all && git reset --hard origin/main`.
- **결론**: **오전 중 bfg 우선 실행**, 이후 작업 착수.

### 2. LLM 실 API 검증 순서
- **Node Dev**: OpenAI(GPT-4o) → Claude → DeepSeek 순서 제안. API 키 Secret 선등록 필요.
- **AI Engineer**: 10턴+ 완주 기준, 비용 추적(quota:daily) 병행 권장.
- **PM**: 비용 리스크(CR-01) 높음 → BL-S4-005(비용 추적) 선행 후 검증 착수.
- **결론**: **비용 추적 Redis 연동 → API 키 등록 → OpenAI 순으로 진행**.

### 3. 오늘의 핵심 목표
- **P0 마무리**: bfg 정리(BL-P0-003) + API 키 로테이션(BL-P0-004) → P0 5건 전량 완료
- **설계 구현 착수**: 퇴장/기권(7 SP) + 중복방(3 SP) + beforeunload(2 SP) = 12 SP
- **LLM 기반 준비**: 비용 추적 설계 + API 키 Secret 등록

### 4. 설계 구현 우선순위
- **Architect**: 퇴장/기권이 최우선 — "좀비 게임" 방지가 가장 큰 GAP.
- **Go Dev**: 퇴장/기권(7) → 중복방(3) → 교착(5) 순서 제안.
- **Frontend Dev**: beforeunload(2)는 독립 작업, 퇴장/기권과 병렬 가능.
- **결론**: **Go Dev 퇴장/기권 + Frontend beforeunload 병렬 진행**.
