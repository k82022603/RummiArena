## 논의 사항

### 1. v2 baseline 재정립 — "얻어걸린 데이터였을까?"

R4 (2026-04-06) v2 30.8% + R5 Run 3 v2 30.8%. 오늘 재실측 **25.6%**. Δ=-5.2%p. 가능한 해석:
- (a) N=1 자연 분산 범위 — DeepSeek 의 자체 variance 가 크다 (R2 5% 선례 존재)
- (b) R4/R5 가 **상향 노이즈**였을 가능성 — "v2=30.8% stable baseline" 믿음이 흔들림
- (c) 게임 상태 초기 배치에 따른 fluctuation

**결론 보류**. v2 multi-run (N=3) 가 근본 해결이지만 이번 Sprint 에는 시간 부족. **Day 7 baseline 으로는 25.6% 를 쓰는 게 더 공정**.

### 2. v4 regression 판정 재검증 — "Day 5 결론을 의심할 때"

재실측 25.6% ≈ v4 Phase 2 25.95% (Δ=+0.35%p, 측정 오차 내). Day 5 "v4 regression" 판정은 **구 baseline 30.8%** 에 의존했음. 오늘 baseline 에서 보면:
- v4 는 regression 이 아닐 수 있음
- Phase 2 Run 1+2 의 fallback (T46 710s) 은 timeout 제약 문제였을 가능성
- **v4 unlimited timeout 실측의 의미가 강화됨** — 정확한 v4 잠재력 검증 필수

### 3. v2-zh 가설 — "중문 프롬프트는 성능 개선 아닌 latency 축약"

공정 A/B (v2 재실측 25.6% vs v2-zh 23.1%) Δ=-2.5%p — 이전 R4 기준 -7.7%p 대비 훨씬 작다. Latency 는 v2-zh 146.7s << v2 203.3s. 해석:
- **번역 오버헤드 제거**는 latency 측면에서 **확증** (v2-zh 가 명확히 빠름)
- **Place rate 개선**은 noise 수준. 중문 프롬프트가 **reasoning 을 축약**시켜 품질 저하 동반
- DeepSeek-R1 이 영문 프롬프트를 읽고 중문 reasoning 으로 사고해도 **자연스러움** — 이중 번역의 loss 가 예상보다 작음
- **결론**: v2-zh production 전환 불필요. v2 유지 (또는 Round 10 에서 v2 N=3 multi-run 로 재검증)

### 4. SKILL 15건 개정 — "SKILL 이 자기 자신을 개정한 drop"

오늘 Architect 가 자신의 SKILL 을 직접 편집 (code-fix +146줄 / code-modification +88줄). 핵심:
- P0-A1 SSOT 매핑 섹션 (41/42/31 문서 계획서 필수 포함)
- P0-A2 §4.4 롤백 준비 (3가지 필수 항목)
- P0-A3 SKILL 진화 트리거 (3가지 조건)
- P0-Q3 재수정 루프 상한 2회 + 에스컬레이션
- P1-Q4 Phase 1.5 QA 계획서 승인 체크
- Mermaid 분기 확장 (QA FAIL: 구현 오류 vs 계획서 결함)

**즉시 효과**: 같은 날 오후 Architect 가 v4 unlimited 계획서 작성 시 신규 SSOT 매핑 + 롤백 섹션을 자기 자신 준수. 자기 참조 고리가 닫힌 최초 사례.

### 6. 자율 실행 원칙과 "앞서나가기" 의 경계

**두 원칙 충돌 시 해석**:
- 자율 실행 (feedback_no_confirmation_prompts) = Y/N 묻지 말고 진행
- 앞서나가지 않기 (오늘 신규) = 현재 지시 단위로만, 다음 단계 선제 계획/실행 금지

**해결**: 현재 지시된 **범위 내**에서 Y/N 없이 실행. 범위 밖은 건드리지 말 것. 오늘 Architect 에게 v4 계획서 선제 위임 → 범위 밖 확장. 계획서는 보관만 하고 이후 사용자 승인 시 활용.

### 7. Agent 모델 정책 개편의 자기참조 — Opus 4.7 xhigh 가 이 스크럼을 쓰고 있다

애벌레 지시로 오전에 메인 세션 + 5 에이전트 (architect/ai-engineer/security/pm/qa) 를 Opus 4.7 xhigh 로 명시. **이 스크럼 로그 자체가 Opus 4.7 xhigh 메인 세션의 산출물**. Sonnet 4.6 다운시프트 5명 (go-dev/node-dev/frontend-dev/devops/designer) 은 오늘 실 작업에서 품질 저하 관찰되지 않음 (Node Dev v2-zh 구현 PASS, DevOps 배포 안정). 정책 유효성 1일차 확인.
