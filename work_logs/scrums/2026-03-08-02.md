## 논의 사항
- WSL 멈춤 원인: `@thelord/mcp-server-docker-npx`가 프로세스 876개를 fork → 메모리 전부 소진
  - 12GB로 올려도 죽은 건 메모리 부족이 아니라 프로세스 버그가 원인이었음
- docker MCP 삭제 결정: Bash `docker` 명령으로 동일 기능 수행 가능, MCP 오버헤드 제거
- .wslconfig: 버그 해결 후 8GB→10GB로 재상향, processors 4→6
- 인프라 문서 필요성: 로컬 환경 구성과 개발 셋업을 체계적으로 정리해둬야 함
  - 로컬 인프라 가이드 (리소스 전략, 교대 실행, 트러블슈팅)
  - 개발 환경 셋업 매뉴얼 (온보딩 체크리스트)
