- **반성**: **BUG-UI-009 는 내가 만든 버그다**. 정확히는 "내가 고치지 않고 덮어두었던 버그들이 v-13e 패치를 계기로 한꺼번에 터진 것". `handleDragEnd` 는 지금 1148줄 파일 안에서 500줄 가까이 되는 함수고 조커·재배치·일반 드래그가 중첩 if/else 로 얽혀 있다. re-entrancy guard 없이 스프린트마다 분기를 하나씩 추가해왔다 — 드래그가 빠르게 연속되거나 취소 이벤트가 끼어들면 상태가 꼬이는 게 구조적으로 예정되어 있었다. 알면서 "지금은 괜찮다" 고 미뤘다. `PlayerRack.tsx:154` key 문제도 — 손패에 같은 코드 타일이 두 장 있는 게 루미큐브 기본인데 code 만으로 key 를 쓴 건 React 기본 규칙 위반. `onDragCancel` 미등록은 변명이 없다. dnd-kit 공식 문서에 명시된 핸들러다. Playwright spec 을 **버그 수정 이후**에 쓰는 관행도 내가 만든 나쁜 선례 — 재현 spec 없으니 회귀 탐지 없고 핫픽스가 또 다른 버그를 낳아도 배포 전에 안 잡힌다. 사용자가 "고쳐진 게 하나도 없는 듯" 이라고 말했을 때 할 말이 없다. 그 말이 맞다. 오늘부터 **재현 spec 작성 → 구현 → spec 통과** 순서 의무화, `handleDragEnd` 를 조커/재배치/일반 3개 독립 핸들러로 분리 + 각각 진입 guard. 이번 주 안에.

## 논의 사항

1. **merge gate 강화** (pm + devops + qa 합의)
   - smoke 정의 상향: 기존 `/health 200 + helm list` → 신규 "**1 게임 완주 + 재배치 4유형 + 한글 렌더 diff + drag stuck 재현 0건**"
   - `pre-deploy-playbook` 스킬 실행 증거를 머지/태그 전에 필수 첨부
   - rollback 임계치 기준 문서화 (devops 주관, 오늘 중)

2. **핫픽스 중첩 탐지** (architect 주관)
   - PR 리뷰에 **구조 부채 누적 카운터** 도입
   - 동일 함수/파일 3회 이상 핫픽스 누적 시 리팩터 ADR 강제
   - `handleDragEnd` 484줄은 frontend-dev 가 이번 주 안에 3개 핸들러로 분리

3. **룰-UX 동기화 SSOT 신설** (ai-engineer + designer 공동 제안)
   - 게임 룰(V-04/V-13a/V-13e 등) 이 **AI 프롬프트 / 서버 에러코드 / 프론트 안내문구 / E2E 재현 시나리오** 4축에서 어떻게 표현되는지 단일 표
   - `docs/02-design/43-rule-ux-sync-ssot.md` (가제) 초안 금주 완성

4. **WS 권한 검증 보안 케이스 추가** (security + qa + go-dev 공동)

   - BUG-UI-011 / BUG-UI-014 에 대한 보안 축 재현 스펙 추가

5. **UX 타이머 레지스트리** (node-dev 주관)
