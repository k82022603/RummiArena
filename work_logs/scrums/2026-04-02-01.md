## 논의 사항

### 1. SonarQube OOM 해결 전략 (어제 합의 재확인)
- **DevOps**: JVM 힙 512m→256m 축소 우선 시도. 실패 시 Runner Pod 메모리 512Mi→1Gi 증가.
- **Architect**: 16GB 장비 제약 고려, 256m으로 충분할 것으로 판단. SonarQube 서버 자체는 Docker Compose이므로 별도 메모리 풀.
- **결론**: 어제 합의대로 진행. 256m 축소 우선.

### 2. Human+AI 플레이테스트 구성
- **QA**: Human(애벌레) 1 + AI 3 구성. GPT(gpt-5-mini) + Claude(claude-sonnet-4) + DeepSeek(deepseek-reasoner) 또는 Ollama(qwen2.5:3b).
- **AI Engineer**: DeepSeek는 Round 3에서 별도로 테스트하므로, 플레이테스트는 GPT + Claude + Ollama 3인이 적절.
- **Frontend Dev**: K8s 서비스 7개 Running 확인 후 진행. 게임 중 UI 이슈 실시간 대응.
- **결론**: **Human(애벌레) + GPT + Claude + Ollama로 4인 대전**, GAME_OVER까지 완주.

### 3. 전 파이프라인 완주 시 의미
- **PM**: lint(4) + test(2) + quality(2) + build(4) + update-gitops(1) = **13/13 PASS** 달성 시 RummiArena 최초의 완전 자동화 파이프라인.
- **Security**: 모든 단계에 보안 게이트 포함 — SonarQube(코드 품질) + Trivy(CVE) + 이미지 스캔(CRITICAL 차단). DevSecOps 완성.
- **DevOps**: GitOps repo에 SHA 자동 업데이트 → ArgoCD 자동 동기화까지 이어지면 진정한 GitOps.
