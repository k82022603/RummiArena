# 📘 Argo CD Admin 비밀번호 재설정 및 접속 매뉴얼

## 📌 개요

본 문서는 Kubernetes 환경에서 **Argo CD 초기 비밀번호 Secret이 존재하지 않는 경우**,
관리자(admin) 계정 비밀번호를 재설정하고 UI에 접속하는 방법을 설명한다.

---

## ⚠️ 문제 상황

다음 명령어 실행 시 오류 발생:

```bash
argocd admin initial-password -n argocd
```

```bash
secrets "argocd-initial-admin-secret" not found
```

---

## 🔍 원인 분석

* Argo CD 설치 후 일정 시간이 지나면
  `argocd-initial-admin-secret`은 자동 삭제됨
* 따라서 초기 비밀번호 조회 방식은 더 이상 사용 불가

---

## 🛠 해결 절차

### 1️⃣ 기존 admin 비밀번호 확인 시도

```bash
kubectl -n argocd get secret argocd-secret -o jsonpath="{.data.admin.password}" | base64 -d
```

📌 결과:

* 값이 출력되지 않거나
* bcrypt 해시 (`$2a$...`) 형태 → 복호화 불가

👉 **비밀번호 재설정 필요**

---

### 2️⃣ 새 비밀번호 생성 (bcrypt)

```bash
argocd account bcrypt --password RummiArena2026!
```

📌 출력 예시:

```bash
$2a$10$rMv76/i5Bv6gh5lWWImmVeGyCcEAY9DdiQktK66JlcgEH639KF1wW
```

---

### 3️⃣ Kubernetes Secret 패치

```bash
kubectl -n argocd patch secret argocd-secret \
  -p '{"stringData": {
    "admin.password": "$2a$10$rMv76/i5Bv6gh5lWWImmVeGyCcEAY9DdiQktK66JlcgEH639KF1wW",
    "admin.passwordMtime": "'$(date +%FT%T%Z)'"
  }}'
```

📌 결과:

```bash
secret/argocd-secret patched
```

---

### 4️⃣ Argo CD 서버 재시작

```bash
kubectl rollout restart deployment argocd-server -n argocd
```

📌 결과:

```bash
deployment.apps/argocd-server restarted
```

---

### 5️⃣ 포트 포워딩

```bash
kubectl port-forward svc/argocd-server -n argocd 8443:443
```

📌 실행 결과:

```bash
Forwarding from 127.0.0.1:8443 -> 8080
```

---
