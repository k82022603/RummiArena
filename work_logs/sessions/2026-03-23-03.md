### ISS-001 FIXED — ai-adapter 400 에러 해소

**원인**: game-server `ws_handler.go`가 ai-adapter에 다음 값 전송:
- `persona: "Rookie"` (대문자) → ai-adapter `@IsEnum` 400
- `difficulty: "easy"` (비표준) → ai-adapter `@IsEnum` 400

**수정 (`ws_handler.go` handleAITurn)**:
- `strings.ToLower(player.AIPersona)` → `"Rookie"` → `"rookie"`
- `normalizeDifficulty()` → `"easy"` → `"beginner"` 등 표준값으로 변환

**normalizeDifficulty 매핑**:
| 입력 | 출력 |
|------|------|
| easy, 하수, beginner | beginner |
| medium, mid, 중수, intermediate | intermediate |
| hard, 고수, expert | expert |
| (기타/빈값) | beginner |

### ISS-001 재테스트 결과

직접 ai-adapter 호출로 수정 전/후 비교:
- 수정 전 (`"Rookie"/"easy"`) → HTTP 400
- 수정 후 (`"rookie"/"beginner"`) → HTTP 200, action=draw

WS E2E 확인:
- ai-adapter 로그: `persona=rookie`, `difficulty=beginner` 수신 확인

- ELO 업데이트: `oldRating=1000 newRating=1039 delta=19` 정상 처리
- ai-adapter → Ollama 연결 시도, 30s × 5회 timeout → forceAIDraw fallback (Ollama 응답 지연 이슈)

### ISS-003 FIXED — Ollama K8s Pod 배포

**배경**: Windows Ollama 삭제 → `172.21.32.1:11434` 접근 불가

**해결 과정**:
1. WSL2 Ollama v0.18.2 설치 (`~/.local/bin/ollama`, GitHub release binary)
2. `~/.bashrc` OLLAMA_HOST → `http://localhost:11434` 수정 + auto-start 설정
3. Helm chart `helm/charts/ollama` 신규 작성
   - initContainer: bash TCP readiness check + `ollama pull gemma3:1b`
   - PVC `ollama-models` (hostpath 2Gi) 영속 저장
   - ResourceQuota 내: limits cpu=2000m, memory=2Gi
4. K8s 배포 성공 (ollama Running, gemma3:1b PVC 저장)
5. ai-adapter ConfigMap → `OLLAMA_BASE_URL: http://ollama:11434`
6. ai-adapter rollout restart → `POST /move` 실 응답 확인
