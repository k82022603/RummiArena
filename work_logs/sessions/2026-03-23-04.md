## 완료한 작업

### BUG-1: WS 연결 실패 — NEXT_PUBLIC_WS_URL 번들 베이킹 문제

**원인 확인**:
```js
// 14h된 frontend 번들에서 발견
let r = "".concat("ws://game-server:8080", "/ws?roomId=").concat(t);
```
- `NEXT_PUBLIC_WS_URL`은 빌드 타임 베이킹 변수 → K8s 런타임 env 무효
- Dockerfile 기본값 `ARG NEXT_PUBLIC_WS_URL=ws://game-server:8080` 사용 중이었음
- 브라우저에서 접근 불가한 클러스터 내부 DNS → WS 연결 불가

**수정**:
- `docker build --build-arg NEXT_PUBLIC_WS_URL=ws://localhost:30080 -t rummiarena/frontend:dev .` 재빌드
- `kubectl rollout restart deployment/frontend -n rummikub`
- 번들 내 `ws://localhost:30080` 확인 후 배포

### BUG-2: NextAuth `/api/auth/csrf` 404 — 리라이트 충돌

**원인 확인**:
- `next.config.ts` 리라이트: `source: "/api/:path*"` → game-server 전체 프록시
- NextAuth 내부 경로(`/api/auth/*`)도 game-server로 프록시 → 404

**수정** (`next.config.ts`):
```typescript
// Before
source: "/api/:path*",
destination: `${gameServer}/api/:path*`,

// After
source: "/api/((?!auth).*)",  // /api/auth/* 제외
destination: `${gameServer}/api/$1`,
```

### BUG-3: 연습 모드 타일 그룹화 불가

**원인 확인**:
1. **타입 기본값 오류**: 신규 그룹 기본값 `type: "run"` → Stage 1(goal=group)에서 validateBoard 에러 → isAllValid=false → 확정 비활성화
2. **드롭 동작 오류**: 빈 보드 드롭 시 항상 새 그룹 생성 → 1타일짜리 그룹 여러 개 생성 → validated 그룹 없음 → 클리어 뱃지 미표시

**수정** (`PracticeBoard.tsx`):
- 신규 그룹 기본 타입: `goal === "run" ? "run" : "group"`
- 빈 보드 드롭 시: 기존 그룹이 있으면 마지막 그룹에 추가, `forceNewGroup` 플래그 있을 때만 신규 생성
- **"+ 새 그룹" 버튼** 추가: 복합 스테이지(Stage 5/6)에서 그룹 분리 시 클릭
