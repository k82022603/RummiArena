## 주요 결정 사항

1. **AI_ADAPTER_URL 미설정 시 graceful 비활성화**: aiClient = nil → broadcastTurnStart에서 AI 턴 스킵. Human-only 게임에 영향 없음
2. **AI 턴 폴백 전략**: GenerateMove 에러 또는 배치 검증 실패 → 무조건 forceAIDraw. 게임이 절대 멈추지 않도록 보장
3. **CharacterService 옵셔널 주입**: @Optional() 데코레이터로 하위 호환 유지. 구 템플릿 → 신 템플릿 점진적 전환 완료
4. **MVP 완료 기준 충족**: "Human 2명이 WebSocket으로 게임 가능" — ws_multiplayer_test.go 7개 테스트로 검증
5. **FinishRoom no-op 설계**: FINISHED/CANCELLED 상태에서 중복 호출 시 조용히 반환. 재연결 등 예외 상황에서 안전
6. **관리자 토큰 3단계 fallback**: NEXT_PUBLIC_ADMIN_TOKEN → sessionStorage → dev-login. 개발/운영 환경 모두 대응

### CI 수정 — test-go sed 순서 오류 (`a5f8fd1`)

**문제**: `sed`가 `go tool cover -func` 전에 실행 → 경로 변환 후 `go tool cover` 소스 파일 찾기 실패 → `script_failure`
**수정**: `go tool cover -func=coverage.out` 먼저 실행 후 `sed` + 리다이렉트 방식으로 변경

```yaml
- go tool cover -func=coverage.out          # 먼저 실행
- sed "s|github.com/.../game-server/|src/game-server/|g" coverage.out > coverage_sonar.out
- mv coverage_sonar.out coverage.out         # 덮어쓰기
```

### CI 최종 수정 — gocover-cobertura 경로 분리 (`2849bb3`)

**문제**: `mv coverage_sonar.out coverage.out` 이후 `gocover-cobertura`가 변환된 경로로 소스를 열려 해서 실패
- `gocover-cobertura`는 경로를 실제 파일로 열어 소스 라인 수를 읽음
- `src/game-server/internal/...` 경로 → CWD(`src/game-server/`) 기준으로 `src/game-server/src/game-server/internal/...` 탐색 → 파일 없음

**수정**: `coverage.out`(원본 module 경로)과 `coverage_sonar.out`(변환된 경로) 영구 분리
- `gocover-cobertura < coverage.out` — 원본 경로로 XML 생성
- `sed ... > coverage_sonar.out` — 덮어쓰기 없이 별도 파일 유지
- `sonar.go.coverage.reportPaths=src/game-server/coverage_sonar.out` 으로 변경
