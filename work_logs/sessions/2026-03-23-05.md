### WS GAME_OVER 400바이트 절단 수정 (go-dev 에이전트)

**원인 3개:**
1. **주요**: `http.Server.WriteTimeout=15s` → WebSocket Hijack 후에도 HTTP deadline 잔존 → 게임 15초 초과 시 GAME_OVER write 중단
2. **구조적**: `BroadcastToRoom`이 동일한 `*WSMessage` 포인터를 4개 커넥션에 공유 → data race
3. **부가**: `Upgrader.WriteBufferSize` 기본 4096 → STALEMATE 4인 1,300bytes 근접

**수정:**
- `cmd/server/main.go`: `WriteTimeout: 15s` → `0`
- `ws_handler.go`: `WriteBufferSize: 8192` 명시
- `ws_connection.go`: `Send()`에서 값 복사본으로 marshal (data race 제거)

**빌드/vet:** PASS
