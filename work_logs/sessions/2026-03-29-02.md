### 1. Google OAuth 로그인 장애 복구 (3번째 반복 — 구조적 해결)

**원인 분석**
- K8s 2026-03-28 복구 시 Google OAuth 자격증명 초기화됨
- ArgoCD sync 시 frontend-config ConfigMap의 GOOGLE_CLIENT_ID가 빈 값으로 덮어쓰임
- game-server-secret의 GOOGLE_CLIENT_ID도 0 bytes → GoogleLoginByIDToken 503 반환

**수정 사항**
- `scripts/inject-secrets.sh` 개선:
  - `src/frontend/.env.local`에서 GOOGLE_CLIENT_ID/SECRET 자동 로드
  - game-server-secret에도 동시 패치 (이전엔 frontend만 주입)
  - kubectl rollout restart 자동 포함
- `argocd/application.yaml`:
  - frontend-config ConfigMap의 GOOGLE_CLIENT_ID를 ignoreDifferences에 추가
  - 이후 ArgoCD sync 시 해당 필드 덮어쓰기 방지

**검증**
- http://localhost:30000/login Google 로그인 정상 동작 확인

### 2. 멀티플레이 GameClient 그룹 병합 버그 수정 (BUG-G-001)

**원인**: `GameClient.tsx handleDragEnd`가 항상 새 그룹 생성, 기존 그룹에 타일 추가 로직 없음

**수정 사항**
- `forceNewGroup` state 추가
- `handleDragEnd` 3단계 분기:
  1. 기존 pending 그룹 영역에 드롭 → 해당 그룹에 타일 추가
  2. 보드 영역 + 마지막 pending 그룹 있음 + forceNewGroup=false → 마지막 그룹에 추가
  3. 그 외 → 새 그룹 생성
- `currentTableGroups`, `currentMyTiles` useMemo 적용 (매 렌더링 새 참조 생성 방지)
- "+ 새 그룹" 버튼 추가 (forceNewGroup 플래그 토글)
