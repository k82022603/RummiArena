## 발생 이슈

### ISS-01: Runner 토큰 403 Forbidden (해결)

첫 2개 glrt- 토큰이 이전 설치 시도 과정에서 소비됨. 새 토큰 발급 후 API 사전 검증 절차 확립.

```bash
# 토큰 유효성 사전 검증 (설치 전 필수)
curl -s -X POST "https://gitlab.com/api/v4/runners/verify" \
  -H "Content-Type: application/json" \
  -d '{"token": "glrt-xxxxxxxxxxxx"}' | python3 -m json.tool
```

### ISS-02: Helm --set 토큰 파싱 오류 (해결)

`--set runnerToken=glrt-xxx.01.yyy`에서 `.`이 Helm 경로 구분자로 해석. values 파일 방식으로 전환.

### ISS-03: golangci-lint Go 버전 불일치 (해결)

golangci-lint v1.62는 Go 1.23 기반. go.mod가 Go 1.24를 요구하여 `requires Go >= 1.24` 오류. v2.1로 업그레이드.

### ISS-04: gocover-cobertura PATH 미포함 (해결)

CI 환경에서 `go install`이 `$GOPATH/bin` 설치. PATH에 없어 `command not found`. `GOBIN=/usr/local/bin` 지정으로 해결.

### ISS-05: sonar-scanner-cli:latest v2 API 비호환 (해결)

SonarQube 9.9 LTS는 `/api/v2/` 미지원. scanner-cli:latest가 v2 API를 사용하여 오류. `5.0` 버전으로 고정.

### ISS-06: update-gitops YAML 파싱 오류 (해결)

`>` fold 스타일 + 큰따옴표가 GitLab YAML 파서에서 멀티라인으로 오해. 단일 따옴표 래핑으로 해결. `glab ci lint` API로 사전 검증 방법 확립.

### ISS-07: trivy-fs CVE 4건 추가 발견 (해결)

1차 수정 후 재스캔에서 신규 CVE 4건 발견. next 15.2.6에 새 GHSA 2건, multer 2.1.0에 CVE-2026-3520, golang.org/x/crypto에 CVE-2025-22869. 연쇄적으로 수정. trivy-fs는 픽스 후 반드시 재스캔 필요.

### ISS-08: sonarqube job 공유 런너 실행 (해결)

tags 미설정으로 GitLab.com 공유 런너와 경쟁. 공유 런너는 `host.docker.internal:9001` 접근 불가 → SonarQube 연결 실패. YAML 앵커 `.local-runner`로 7개 job 일괄 처리.
