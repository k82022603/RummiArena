## 메모

### 인사이트 1 — LLaMA 소형 모델의 한계와 우회
qwen2.5:3b 같은 3B 파라미터 모델은 13개 타일에서 멜드 조합을 자체 탐색하지 못한다. v8-ollama-place 프롬프트는 게임 엔진이 사전 계산한 멜드 후보를 주입하여 모델의 역할을 "조합 탐색"에서 "후보 선택"으로 좁혔다. 이 패턴은 다른 소형 모델에도 일반화 가능. GPU + Think 모드 가설은 후속 실험 대상 (`docs/04-testing/106`).

### 인사이트 2 — confirmBusy 영구 잠금 패턴
`!draftPendingTableGroups` 같은 truthy/falsy 검사가 객체 lifecycle 의 일부 단계만 커버할 때, 나머지 단계에서 잠금이 영원히 풀리지 않는 패턴. 단순한 boolean 조건이 아니라 **상태 전이 그래프** 관점에서 unlock 경로를 명시해야 한다. 회귀 가드 3건은 이 관점을 테스트로 박제.

### 인사이트 3 — Option C 의 가치
GameClient 의 109줄 handleConfirm 을 `useTurnActions` 5줄로 줄인 것은 라인 수 감소가 본질이 아니라, **dispatcher(GameClient) 와 도메인 로직(hook) 의 책임 분리**가 본질. 조커 갭 false positive 가 부수적으로 해소된 것은 검증 경로가 두 군데에 흩어져 있을 때 한쪽이 stale 해진 결과였다. 단일 진입점이 곧 단일 진실 원천(SSOT).
