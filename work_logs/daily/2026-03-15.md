## 오늘 한 일

### 인프라: SonarQube 기동 완료

SonarQube lts-community 컨테이너를 정상 기동했다. 핵심 문제는 Elasticsearch bootstrap check 실패였다. JVM 힙 설정에서 `Xms ≠ Xmx`이면 Elasticsearch가 시작을 거부한다. `docker-compose.cicd.yml`의 `SONAR_SEARCH_JAVAOPTS`를 `-Xms256m -Xmx512m`에서 `-Xms512m -Xmx512m`으로 수정했다. 단, `docker restart`로는 환경변수가 적용되지 않으므로 전체 `down && up`이 필수였다. 이제 `http://localhost:9001` 에서 `{"status":"UP"}` 응답.

### 게임 서버: 통합 테스트 50개 GREEN

`ConfirmRequest` DTO에 `JokerReturnedCodes []string` 필드를 추가하여 V-07 (조커 반드시 재배치) 규칙이 서비스 레이어에서 engine까지 완전히 전파되도록 연결했다. 신규 테스트 8개:

- **ConfirmTurn 6건**: JokerSwap_Success, JokerSwap_NotReplaced, InitialMeld_ExactlyThirty, InitialMeld_ModifyExistingSet, MultipleValidSets, DrawAction_NoTableChange
- **PlaceTiles 2건**: InvalidSet_ValidationOnConfirm, TableStateSnapshot

기존 42개 + 신규 8개 = 총 50개, 전체 GREEN.

### 프론트엔드: 실시간 UX 2가지

**PLACE_TILES 프리뷰**: 타일을 보드에 드롭해도 즉시 서버에 전송하지 않는다. `gameStore`의 `pendingGroupIds: Set<string>`에 등록만 해두고, `GameBoard`에서 해당 그룹을 반투명(opacity-0.55) + 노란 점선 테두리로 "미확정" 표시한다. 사용자가 "확정" 버튼을 눌러야 `PLACE_TILES` WebSocket 메시지가 전송된다. 실수 방지와 UX 향상 두 가지를 동시에.

**INVALID_MOVE 에러 UI**: 서버에서 `INVALID_MOVE` 메시지가 오면 에러 코드를 한글로 매핑한다. `ERR_INITIAL_MELD_SCORE` → "초기 제출 점수가 30점 미만입니다 (최소 30점 필요)" 등 4종. `ErrorToast` 컴포넌트(3초 자동 소멸)를 활용한다.

### ISS-02: glab auth --hostname 함정

| 항목 | 내용 |
|------|------|

### ISS-03: git remote set-url --add --push 순서 문제

| 항목 | 내용 |
|------|------|
| 심각도 | Low |
| 상태 | 해결, 가이드 반영 |

첫 실행 시 기존 GitHub push URL이 대체됨. GitHub URL 재추가 필요. 가이드에 명시.

### ISS-04: Docker bridge 네트워크로 SonarQube 접근 불가

| 항목 | 내용 |
|------|------|
| 심각도 | Low |
| 상태 | 해결 |

SonarQube가 `rummiarena-cicd_cicd-net`에 있어 기본 bridge IP로 접근 불가. `--network host` 모드로 해결.

### ISS-05: SonarQube 커버리지 경로 불일치

| 항목 | 내용 |
|------|------|
| 심각도 | Low |
| 상태 | 해결 |

프로젝트 루트 마운트 시 coverage.out의 Go 모듈 경로와 소스 경로 불일치 → `src/game-server` 직접 마운트로 해결.

### ISS-06: GitLab push 토큰 만료

| 항목 | 내용 |
|------|------|
| 심각도 | Low |
| 상태 | 미해결 (수동 조치 필요) |

glab config의 `glpat-...` 토큰 만료로 GitLab push 실패. GitHub push는 정상.
→ GitLab 웹 UI에서 PAT 재발급 후 `glab auth login --hostname gitlab.com` 재실행 필요.

### ISS-07: WSL2+SWC Turbopack 로컬 빌드 패닉

| 항목 | 내용 |
|------|------|
| 심각도 | Low |
| 상태 | 우회 해결 |

`npm run build` 시 SWC가 WSL2 마운트 경로 resolve 실패 (known bug). TypeScript 타입 체크는 정상. Docker 빌드는 정상. 로컬 빌드 대신 Docker build 사용 유지.
