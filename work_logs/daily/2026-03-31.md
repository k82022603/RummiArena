### 세션 #01: LLM 4종 API 검증 + 모델 업그레이드 + 프롬프트 고도화

**LLM 4종 실 API 검증 (BL-S4-001~003)**
- K8s ai-adapter-secret에 API 키가 비어있는 문제 발견 — ArgoCD selfHeal이 빈 값으로 되돌리고 있었음
- 원인: argocd/application.yaml에 ai-adapter-secret ignoreDifferences 누락
- ignoreDifferences 추가 후 API 키 재주입 → OpenAI, Claude, DeepSeek, Ollama 4종 전부 API 연결 성공
- 초기 전략 품질 평가: Claude(A) > DeepSeek(C) > OpenAI(D) > Ollama(F)

**프롬프트 최적화 (1차)**
- 초기등록 30점 규칙 강화 (persona.templates.ts, prompt-builder.service.ts)
- 결과: OpenAI D->A, DeepSeek C->B, Claude A->A 유지
- 프롬프트 한 줄의 변경이 모델 2단계 등급 상승을 이끌어냄

**모델 업그레이드**

| 서비스 | 변경 전 | 변경 후 | 사유 |
|--------|---------|---------|------|
| OpenAI | gpt-4o-mini -> gpt-4o | **gpt-5-mini** | 추론 모델, 5~10배 저렴, 전략 품질 우수 |
| Ollama | gemma3:1b | **qwen2.5:3b** | reasoning 양호, K8s 92초 (CPU-only) |
| DeepSeek | deepseek-chat | deepseek-chat 유지 | deepseek-reasoner 테스트했으나 비용 20배, 보수적 전략 |

- gpt-5-mini: reasoning 모델 대응 코드 추가 (max_completion_tokens, temperature 제거)
- qwen3:4b thinking 시도 -> CPU 환경에서 5분+ timeout으로 포기
- thinking 필드 파싱 코드 구현 (향후 GPU 확보 시 활성화 가능)
- deepseek-reasoner reasoning_content 파싱 코드 구현 (환경변수 전환 가능)

**프롬프트 고도화 (2차)**
- tableGroups = "배치 후 테이블 전체 최종 상태" 명확화
- 테이블 재배치 규칙 섹션 신설 (initialMeldDone 분기)
- 전략 분석 절차 5단계 가이드 추가
- 다중 그룹 배치 예시 추가 (예시 C: 63점 6장, 예시 D: 기존 테이블 유지)
- 런 규칙 상세화 (1~13 범위, 순환 불가)

**인프라 변경**
