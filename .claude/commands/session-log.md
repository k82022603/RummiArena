세션 로그를 생성하거나 종료 처리한다.

## 규칙
- 경로: `work_logs/sessions/YYYY-MM-DD-NN.md`
- 세션당 1개. NN은 같은 날짜 내 순번 (01, 02, ...)
- 인자가 없으면: 새 세션 시작
- 인자가 "close" 또는 "종료"이면: 진행 중인 세션 종료 처리

## 새 세션 시작
1. 같은 날짜의 기존 세션 파일 확인하여 다음 순번 결정
2. 현재 시각으로 시작 시간 기록
3. 이전 세션의 "다음 세션 TODO"를 참고하여 목표 초안 작성
4. 사용자에게 목표 확인 요청

## 세션 종료
1. 종료 시간이 (진행 중)인 세션 파일 찾기
2. 대화 컨텍스트, git log를 참고하여 완료한 작업, 이슈, 결정사항 채우기
3. 종료 시간 기록
4. 다음 세션 TODO 작성
5. 사용자에게 검토 요청

## 템플릿
```markdown
# 세션 로그

- **날짜**: YYYY-MM-DD
- **세션**: #NN
- **시작**: HH:MM
- **종료**: HH:MM
- **Phase / Sprint**: Phase X / Sprint N

## 목표
-

## 완료한 작업
-

## 이슈 / 블로커
-

## 주요 결정 사항
-

## 커밋 이력
-

## 다음 세션 TODO
-

## 메모
-
```

## 주의사항
- 톤: 정확하고 상세하게
- 커밋 이력은 `git log --oneline`에서 추출
- 현재 Phase/Sprint 정보는 MEMORY.md 참조
