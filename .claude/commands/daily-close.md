하루 마감 처리를 한다. 로그 마감 + 프로젝트 문서 현행화 + 커밋까지 일괄 수행.

## 실행 순서

### 1. 진행 중인 세션 종료
- 종료 시간이 (진행 중)인 세션 로그를 찾아 종료 처리
- 대화 컨텍스트, git log 기반으로 완료 작업, 이슈, 결정사항 채우기
- 다음 세션 TODO 작성

### 2. 데일리 로그 최종 업데이트
- 오늘의 데일리 로그(`work_logs/daily/YYYY-MM-DD.md`) 읽기
- 오늘 모든 세션 로그를 종합하여 "오늘 한 일" 최종 정리
- "내일 할 일" 작성
- 블로커 업데이트

### 3. 바이브 로그 최종 업데이트
- 오늘 바이브 로그(`work_logs/vibe/YYYY-MM-DD.md`)가 있으면 대화에서 나온 새 아이디어/영감 추가
- 없으면 오늘 대화 전체를 스캔하여 바이브 로그 감성의 아이디어(영감, 실험 가설, 흥미로운 발견 등)가 있었으면 자동 생성
- 해당 내용이 없으면 스킵

### 4. 프로젝트 문서 현행화
오늘 변경된 내용을 반영하여 아래 3개 파일을 자동 갱신한다.

#### PLAN.md
- 체크리스트 상태 업데이트 (완료된 항목 `[x]` 처리)
- "현재 진행 상황" 섹션 갱신
- 새로운 결정 사항이 있으면 "핵심 미결정 사항" 테이블 업데이트

#### README.md (work_logs/README.md)
- 로그 규칙이 변경되었으면 반영
- 새 로그 유형이 추가되었으면 반영

#### CLAUDE.md
- 프로젝트 구조 변경이 있으면 Repository Structure 갱신
- Tech Stack 변경이 있으면 반영
- Key Design Principles 변경이 있으면 반영
- 변경 없으면 건드리지 않음 (불필요한 수정 금지)

### 5. MEMORY.md 업데이트
- "다음 할 일 (Next TODO)" 섹션 갱신
- 완료된 항목 제거, 새 항목 추가

### 6. 변경 사항 요약 + 커밋 + 푸시 (필수)
- 사용자에게 업데이트된 파일 목록과 내용 요약 보여주기
- **확인 없이 즉시 커밋 + push 실행** (데일리 마감 커밋은 필수 절차)
- 커밋 메시지: `데일리 마감 (YYYY-MM-DD)`
- **`commit-push` SKILL Phase 3 준수: origin(GitHub) + gitlab 양쪽 push**
  ```bash
  git push origin main
  git push gitlab main --force   # GitHub이 SSOT, GitLab은 미러
  ```
- 양쪽 push 완료 확인 후 결과 보고

## 주의사항
- **커밋 + push는 데일리 마감의 필수 단계** — 사용자 확인 불필요
- **origin + gitlab 양쪽 push 필수** — 한쪽만 push하면 미완료 (2026-04-26 교훈)
- push 실패 시 원인 안내 (remote 미설정, 충돌, protected branch 등)
- 문서 현행화는 실제 변경이 있을 때만. 변경 없으면 "변경 없음" 표시
- GitLab push 실패 시 `--force` 허용 (GitHub이 SSOT)
