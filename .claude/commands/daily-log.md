오늘의 데일리 로그를 작성/업데이트한다.

## 규칙
- 경로: `work_logs/daily/YYYY-MM-DD.md`
- 하루 1개. 같은 날짜 파일이 있으면 업데이트, 없으면 신규 생성
- 날짜는 오늘 날짜 자동 적용

## 파일이 없을 때 (신규 생성)
1. 오늘 날짜로 파일 생성
2. 대화 컨텍스트와 세션 로그, git log를 참고하여 내용 채우기
3. 사용자에게 검토 요청

## 파일이 있을 때 (업데이트)
1. 기존 파일 읽기
2. 대화 컨텍스트와 세션 로그, git log를 참고하여 변경된 내용 반영
3. 사용자에게 변경점 요약

## 템플릿
```markdown
# 데일리 로그

- **날짜**: YYYY-MM-DD (요일)
- **Phase / Sprint**: Phase X / Sprint N

## 어제 한 일
-

## 오늘 한 일
-

## 내일 할 일
-

## 블로커
-
```

## 주의사항
- 톤: 간결하게
- 현재 Phase/Sprint 정보는 MEMORY.md 참조
- "오늘 한 일"은 세션 로그와 git log에서 추출
