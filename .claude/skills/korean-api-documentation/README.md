# 한글 API 문서화 표준

> 일관되고 명확한 한글 API 문서를 작성하여 개발자 경험(DX)을 향상시킵니다.

## 언제 사용하나

- REST API 엔드포인트 문서를 한글로 작성할 때
- OpenAPI(Swagger) 스펙에 한글 설명을 추가할 때
- 에러 코드 문서, Python docstring 등 기술 문서의 한글 표준이 필요할 때

## 핵심 흐름

1. 필수 6개 섹션 확인: 개요, 인증, 엔드포인트 목록, 상세 명세, 에러 코드, 예제
2. 엔드포인트 템플릿에 맞춰 작성 (Headers, Body, 응답, 에러, cURL)
3. 용어 표준 적용 (외래어 원어 유지, 동사형 종결, 존댓말)
4. 품질 체크리스트 확인 (파라미터 타입/필수/설명, cURL 복사 실행 가능 등)
