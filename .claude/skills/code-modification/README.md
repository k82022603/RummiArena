# 코드 수정 표준 절차

> "생각 없이 고치면, 고친 것이 아니라 옮긴 것이다."

## 언제 사용하나

- 소스 코드를 한 줄이라도 수정하기 전 (모든 Dev 에이전트 필수)
- 설정값(타임아웃, 프롬프트 variant, 에이전트 모델 등) 변경 시
- 긴급 핫픽스 시에도 축약은 가능하되 SSOT 매핑은 생략 금지

## 핵심 흐름

1. **Phase 1 -- 분석**: 수정 대상 Read, caller/callee Grep 전수 검색, 기존 테스트 확인
2. **Phase 2 -- 계획**: 무엇을/왜/어디까지 변경하는지 정리, 위험 체크
3. **Phase 3 -- 구현**: 계획에 있는 것만 수정, 피호출 -> 호출자 순서, 동반 수정 체크
4. **Phase 4 -- 검증**: 빌드 확인, 관련 테스트 실행, 셀프 리뷰, 롤백 준비
