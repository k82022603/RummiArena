## Critical 이슈 상세

### C-1: WebSocket CheckOrigin CSRF 취약점

**위치**: `src/game-server/internal/handler/ws_handler.go:25`

```go
// 수정 전 (취약)
var upgrader = websocket.Upgrader{
    CheckOrigin: func(r *http.Request) bool {
        return true // TODO: 프로덕션에서는 origin 검증
    },
}
```

**위험도**: 악성 웹사이트가 사용자의 브라우저를 통해 게임 서버 WebSocket을 하이재킹 가능.

**수정**: `CORS_ALLOWED_ORIGINS` 환경변수 기반 화이트리스트 검증 적용.

### C-2: Google id_token JWKS 서명 미검증

**위치**: `src/game-server/internal/handler/auth_handler.go:282`

`parseIDTokenPayload()`는 JWT의 Base64 디코딩만 수행하고 Google의 공개키(JWKS)로 서명을 검증하지 않음. 공격자가 조작된 `sub` 클레임을 가진 토큰을 `POST /api/auth/google/token`에 직접 전송하면 임의 사용자로 로그인 가능.

**Phase 2.5에서 `google.golang.org/api/idtoken` 패키지로 수정 예정.**

### C-3: ai-adapter POST /move 인증 없음

**위치**: `src/ai-adapter/src/move/move.controller.ts:163`

`AI_ADAPTER_INTERNAL_TOKEN` 설정은 config에 있지만 컨트롤러에서 검증하지 않음. `X-Internal-Token` 헤더 Guard 추가로 수정.

## Major 이슈 상세

### M-1: Dockerfile NEXT_PUBLIC_API_URL /api 누락

오늘 발생한 `net::ERR_FAILED`의 근본 원인.
- 수정 전: `ARG NEXT_PUBLIC_API_URL=http://game-server:8080`
- 수정 후: `ARG NEXT_PUBLIC_API_URL=http://game-server:8080/api`

### M-2: SERVER_MODE 기본값 debug

Gin debug 모드는 스택 트레이스를 HTTP 응답에 포함. 기본값을 `release`로 변경.

### M-3: Room 인메모리 저장

`repository.NewMemoryRoomRepo()` 사용으로 Pod 재시작 시 모든 방 정보 소실. 아키텍처 원칙("Stateless 서버") 위반. Sprint 3에서 Redis RoomRepo 구현 예정.

### M-4: CORS AllowOrigins 하드코딩

방금 추가된 CORS 미들웨어가 localhost만 허용. `CORS_ALLOWED_ORIGINS` 환경변수로 교체.

### M-5: Rate Limiting 없음

`/api/auth/google/token`, `/api/auth/dev-login` 엔드포인트에 요청 제한 없음. Traefik IngressRoute 레벨 또는 gin-contrib/ratelimit으로 Sprint 3에서 적용 예정.

### M-6: room.Name 입력 검증 없음

```go
// 수정 전
Name string `json:"name"`

// 수정 후
Name string `json:"name" binding:"max=50"`
```

### M-7: Frontend 테스트 전무

게임 엔진(Go, 262개), AI 어댑터(NestJS, 다수)와 달리 Frontend는 테스트 파일 없음. Sprint 3에서 최소 단위 테스트 추가 예정.
