# 58. v4.1 vs v4 vs v2 DeepSeek Reasoner 실측 검증 보고서

- **작성일**: 2026-04-16
- **모델**: deepseek-reasoner
- **검증 목적**: Round 6 Phase 2 N=2 에서 확인된 v4 regression (place 25.95% vs v2 30.8%, avg latency +52%, max +94%) 의 원인이 Thinking Budget 지시 블록인지 single-variable A/B 로 분리 검증
- **스크립트**: `src/ai-adapter/scripts/verify-v4.1-deepseek-empirical.ts`
- **LangSmith**: disabled

## 결론 (TL;DR)

**V4_1_NEUTRAL** — v4 regression 자체가 본 fixture 에서 관찰되지 않음 — Round 6 결과가 noise 였을 가능성

판정 근거:
- latency_ms mean: v2=247451 v4=295658 v4.1=273424
- reasoning_tokens mean: v2=5617 v4=6586 v4.1=6072 | Cohen d(v4 vs v2)=0.68 | Cohen d(v4.1 vs v4)=-0.35
- tiles_placed mean: v2=6.33 v4=7.00 v4.1=6.33 | Cohen d(v4.1 vs v2)=0.00
- completion_tokens mean: v2=7834 v4=9502 v4.1=8657
- A (v4.1 latency <= v2*1.2): PASS  (273424 vs limit 296942)
- B (v4.1 tiles >= v2-0.5):   PASS  (6.33 vs limit 5.83)
- C (v4.1 reasoning <= v4*0.8): FAIL  (6072 vs limit 5269)
- 주의: 본 fixture 에서는 v4 가 v2 대비 유의한 regression 을 보이지 않음. N=3 fixture 검증의 한계일 수 있음.

## 8. 한계

- N=3 single fixture. 통계적 유의성은 제한적이며 "원인 분리 탐침" 목적.
- DeepSeek 는 usage.completion_tokens_details.reasoning_tokens 를 제공하지 않음. reasoning_content 길이 approximation 사용.
- temperature 파라미터 미전송 (어댑터 호환) — stochastic variance 가 존재.
- 본 검증은 single turn 수준이며, Round 6 의 multi-turn 누적 효과 (후반 복잡도 증가) 는 재현하지 않는다.
