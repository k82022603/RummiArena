## 4. 리스크 모니터링

- Sprint 회고 시 리스크 상태 점검
- AI 호출 비용 주간 리뷰
- K8s 리소스 사용량 일일 확인 (`kubectl top`)
- 카카오톡 API 일일 발송량 모니터링
- WSL2 메모리 사용량 모니터링 (`wsl --status`, `.wslconfig` 프로파일 점검)
- 리스크 오너(애벌레)가 Sprint 단위로 리스크 등급 재평가
