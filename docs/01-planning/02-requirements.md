#### FR-004-09 AI 캐릭터 시스템 상세

**캐릭터 (6종)**

| 캐릭터 | 전략 성향 | 설명 |
|--------|-----------|------|
| Rookie | 기본형 | 단순 규칙 기반, 가능한 수를 순차적으로 탐색 |
| Calculator | 계산형 | 확률 기반 최적 수 계산, 기대값 최대화 |
| Shark | 공격형 | 빠른 타일 소진, 상대 견제 우선 |
| Fox | 전략형 | 상황 판단 후 공격/보수 전환, 유연한 플레이 |
| Wall | 보수형 | 안정적 세트 구성, 리스크 최소화 |
| Wildcard | 변칙형 | 비정형 패턴, 예측 불가능한 플레이 |

**난이도 (3단계)**

| 난이도 | 설명 | 프롬프트 복잡도 |
|--------|------|----------------|
| 하수 | 기본 규칙만 준수, 최적화 없음 | 낮음 |
| 중수 | 2~3수 앞 탐색, 기본 전략 적용 | 중간 |
| 고수 | 상대 타일 추론, 최적 전략, 장기 계획 | 높음 |

**심리전 Level (0~3)**

| Level | 설명 |
|-------|------|
| 0 | 심리전 없음 (순수 전략) |
| 1 | 기본 블러핑 (의도적 지연, 페이크 무브) |
| 2 | 상대 패턴 분석 기반 역이용 |
| 3 | 고급 심리전 (타일 홀딩, 역 블러핑, 상대 유인) |

## 3. 우선순위 분류 (MoSCoW)

| 분류 | 설명 | 요구사항 수 |
|------|------|------------|
| **Must** | 반드시 필요. 없으면 프로젝트 실패 | FR-001 게임 엔진 핵심, FR-001B Stage 1~5, FR-002 멀티플레이, FR-003 인증, FR-004 AI 핵심, FR-005 관리자 핵심, FR-007-01 카카오톡, FR-008 UI 핵심, NFR 성능/보안/운영 핵심 |
