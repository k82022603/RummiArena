# 로컬 LLM 비교 분석 (Local LLM Comparison)

## 1. 서빙 도구 비교

Ollama 외에도 로컬 LLM을 서빙하는 도구가 여럿 있다.

| 항목 | **Ollama** | **vLLM** | **llama.cpp** | **LocalAI** | **LM Studio** | **Jan** |
|------|-----------|----------|---------------|-------------|---------------|---------|
| Windows 설치 | 매우 쉬움 | 어려움 (WSL 필수) | 보통 (빌드 필요) | 보통 (Docker) | 매우 쉬움 (GUI) | 매우 쉬움 (GUI) |
| OpenAI 호환 API | O (내장) | O (내장) | O (별도 서버) | O (핵심 기능) | O (내장) | O (내장) |
| K8s 배포 | 가능 (커뮤니티 Helm) | 우수 (프로덕션급) | 수동 구성 | 좋음 (Docker 기반) | 불가 (데스크탑) | 불가 (데스크탑) |
| CPU 전용 성능 | 좋음 | 나쁨 (GPU 전용) | 최고 | 좋음 | 좋음 | 좋음 |
| 메모리 효율 | 좋음 (GGUF) | 나쁨 (GPU VRAM) | 최고 (GGUF) | 좋음 | 좋음 | 좋음 |

### 이 프로젝트 결론
- **개발/실험용**: **Ollama** (설치 간편, API 내장, 모든 모델 지원, K8s 배포 가능)
- GPU 있으면: vLLM 추가 고려
- GUI 원하면: LM Studio 보조 사용

### 2.2 DeepSeek-R1 상세

```
ollama pull deepseek-r1:7b     # Qwen2.5 기반 증류
ollama pull deepseek-r1:14b    # Qwen2.5 기반 증류 (추천)
ollama pull deepseek-r1:32b    # 16GB+ RAM 필요
```

**강점**: 체인-오브-소트(CoT) 추론. 복잡한 타일 재배치 시나리오에서 단계적 사고 가능.
**약점**: 추론 과정이 길어 응답 느림. 작은 크기에서 JSON 형식 준수율 하락.
**게임 AI 적합도**: 고수 캐릭터에 적합. 심리전/재배치 추론에 강점.

### 2.3 Qwen 2.5 / Qwen 3 상세

```
ollama pull qwen2.5:7b         # 범용 (추천)
ollama pull qwen2.5:14b        # 고품질
ollama pull qwen2.5-coder:7b   # JSON 출력 특화
ollama pull qwen3:8b           # 최신, 하이브리드 사고 모드
```

**강점**: JSON 구조화 출력 품질 최고. 지시 따르기 우수. Qwen 3는 사고 모드 온/오프 가능.
**약점**: 대형 모델(32B+) 아니면 DeepSeek-R1급 깊은 추론은 어려움.
**게임 AI 적합도**: 중수~고수 캐릭터에 최적. JSON 안정성이 높아 파싱 에러 적음.

### 2.4 기타 모델 요약

| 모델 | 게임 AI에서의 포지션 |
|------|---------------------|
| **LLaMA 3.2 1B/3B** | 하수 캐릭터 전용. 빠르고 가볍지만 추론 약함. 의도적 실수에 적합 |
| **LLaMA 3.1 8B** | 범용 중수. 안정적이지만 JSON/추론에서 Qwen에 밀림 |
| **Gemma 3 4B/12B** | 균형형. 속도와 품질의 좋은 타협점 |
| **Phi-4 14B** | 크기 대비 최고 추론력. 고수 캐릭터 후보. RAM 9GB 필요 |
| **Phi-4-mini 3.8B** | 작지만 똑똑한 중수. 빠른 응답 |

## 3. 이 프로젝트 권장 모델 조합

### 캐릭터 × 모델 매핑

| 캐릭터 (난이도) | 권장 모델 | RAM | 이유 |
|----------------|-----------|-----|------|
| **Rookie (하수)** | `llama3.2:1b` 또는 `gemma3:1b` | 1GB | 빠르고, 자연스러운 실수 |
| **Wall (중수, 수비)** | `phi4-mini` 또는 `qwen2.5:3b` | 2.5GB | 빠른 응답, 적당한 전략 |
| **Calculator (중수, 논리)** | `qwen2.5:7b` 또는 `qwen3:8b` | 5GB | JSON 정확도 높음, 논리적 |
| **Wildcard (중수, 예측불가)** | `gemma3:4b` | 3GB | 다양한 응답 패턴 |
| **Shark (고수, 공격)** | `qwen2.5:14b` 또는 `phi4` | 9GB | 강한 전략 + 정확한 JSON |
| **Fox (고수, 교활)** | `deepseek-r1:14b` | 9GB | 체인 추론으로 심리전 시뮬레이션 |

### 동시 실행 시 메모리 예상

| 시나리오 | 모델 조합 | 총 RAM |
|----------|-----------|--------|
| 하수 1 + 중수 2 | 1b + 3b + 7b | ~8.5GB |
| 중수 2 + 고수 1 | 3b + 7b + 14b | ~16.5GB |
| 고수 3 (토너먼트) | 14b × 3 (순차 로딩) | ~9GB (Ollama 자동 교체) |

> Ollama는 기본적으로 모델을 순차 로딩/언로딩하므로, 가장 큰 모델 크기 + 여유분만 있으면 됨.

## 4. 설치 퀵스타트

```bash
# Ollama 설치 (WSL2)
curl -fsSL https://ollama.com/install.sh | sh

# 현재 장비(RAM 16GB) 기준 권장 모델
ollama pull llama3.2:1b       # 하수용 (~700MB, ~1GB RAM)
ollama pull qwen2.5:3b        # 중수용 (~2GB, ~3GB RAM)
ollama pull phi4-mini          # 고수용 (~2.5GB, ~4GB RAM)
# 주의: 7B 이상 모델은 K8s와 동시 실행 불가 (AI 실험 모드에서 단독 실행)

# 테스트
ollama run qwen2.5:7b "JSON으로 응답하세요: {action: place 또는 draw}"
```

## 5. API 비용 vs 로컬 모델 비교

| 항목 | API 모델 (GPT-4o, Claude) | 로컬 모델 (Ollama) |
|------|--------------------------|-------------------|
| 비용 | 호출당 과금 | 0원 |
| 속도 | 2~5초 | 3~30초 (CPU) |
| 품질 | 최상 | 중~상 (모델 크기 의존) |
| 오프라인 | 불가 | 가능 |
| 토너먼트 (수백 게임) | 비용 폭증 | 무료 |
| JSON 안정성 | 매우 높음 | 모델 의존 (Qwen 높음) |
